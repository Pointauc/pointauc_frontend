# Settings Page Design Brief

## Purpose

This page is the main settings area for the product. It should let users configure the auction system, connect external services, manage reusable setting templates, and access a few advanced tools.

The new design does not need to preserve the current layout, but it should preserve the full content and all major controls.

## What Needs To Be Present

### Page Identity

- A settings template selector
- Multiple different sections to group settings logically

### Settings Templates

Users need a template/profile system for settings.

### Auction Settings Content

#### Timer Settings

Include controls for:

- showing total auction time
- adding time when the leader changes
- adding time when a new position appears
- adding time when a donation happens
- limiting additional time based on a timer threshold

This area should feel like a focused block of timing rules and numeric controls.

#### Appearance Settings

Include controls for:

- custom auction background
- changing or removing background
- adjusting overlay strength over the background
- adjusting background blur
- choosing a main accent color
- resetting colors to default values

This section should support a design language around customization and branded atmosphere.

#### Auction Display Settings

Include controls for:

- showing total sum
- showing winning chances
- hiding lot and bid amounts

This area is about how auction information is presented to users.

#### State Transfer Tools

Include:

- export settings/state
- import settings/state

This should read as an advanced tools area.

#### Optional Feature Blocks

Include separate configurable blocks for:

- marbles auction
- lucky wheel

These should feel like optional modules that can be enabled and configured.

### Integration Settings Content

#### Common Integration Settings

Include:

- auto-add bids setting
- bid queue sorting setting
- bid naming mode
- API documentation link
- API token access
- token reveal / copy / regenerate actions

This section should communicate technical configuration and external access.

#### Channel Points / Rewards

Include:

- connection state for points-based integrations
- connect / disconnect actions
- controls related to rewards handling
- option to return canceled bids
- option to bind rewards to timer behavior
- editable reward presets list

Reward presets need:

- shared naming/prefix field
- list of reward entries
- cost field
- color field
- add and remove actions
- save action

This section should support a more structured, data-heavy editing experience.

#### Donation Integrations

Include:

- available donation providers
- connected provider states
- connect / disconnect actions
- donation-to-points conversion settings
- reverse conversion direction control
- example conversion display
- option to add time on donation
- numeric value for added time

This section should communicate connections, monetization, and conversion setup.

## Visual Priorities

The page should feel like a professional control panel for a live auction product.

Key qualities to design for:

- clear hierarchy between major groups
- strong separation between basic settings and advanced/technical tools
- easy scanning across many controls
- room for both simple toggles and denser configuration blocks
- clear distinction between reusable templates, integrations, and product settings
- support for modules that may be enabled, disabled, connected, or unavailable

## Important Content Characteristics

- The page mixes simple settings, structured forms, external service management, and advanced tools.
- Some sections are lightweight and preference-based.
- Some sections are denser and closer to dashboard/configuration interfaces.
- The design should handle both individual controls and grouped editing surfaces.
- The design should leave space for status messaging such as connected, disconnected, active, unavailable, or sensitive.

## Short Summary

Design this as a settings hub for an auction platform with three main content types:

- product behavior settings
- integration and account connection settings
- reusable templates and advanced tools

It should support customization, technical configuration, and structured data editing within one coherent page.
